# フロントエンド実装ガイド：購入・ダウンロード機能

## 概要

本ドキュメントでは、AIコンテンツマーケットにおける「購入者だけがコンテンツをダウンロードできる機能」をフロントエンドで実装するための一連の流れを記載します。

---

## 前提条件

- バックエンドAPIは `http://127.0.0.1:8000` で起動している
- 全リクエストのヘッダーに `Content-Type: application/json` を付ける
- 認証が必要なリクエストは `Authorization: Bearer {token}` を付ける

---

## STEP 1: ログイン機能の実装

### 目的
ユーザーを認証し、以降のリクエストで使用するトークンを取得する。

### 手順

1. ログイン画面にメールアドレスとパスワードの入力フォームを用意する
2. 「ログイン」ボタンを押したら `POST /api/login` にメールアドレスとパスワードを送信する
3. レスポンスに含まれる `token` を取得し、アプリ内（例：localStorageやstateなど）に保存する
4. ログイン失敗時（メールアドレスまたはパスワードが誤り）はエラーメッセージを表示する

### 補足
- まだアカウントがないユーザー向けに `POST /api/register` で新規登録も同様に実装できる
- 登録・ログインどちらも成功すると同じ形式で `token` が返ってくる
- token はアプリを通じて使い回すため、ログアウトするまで保持し続ける

---

## STEP 2: 商品一覧の取得・表示

### 目的
販売中の商品を一覧で画面に表示する。

### 手順

1. ページ読み込み時に `GET /api/products` を叩く
2. 返ってきた商品の配列を画面に表示する
3. 各商品について以下の情報を表示する
   - タイトル（`title`）
   - 作者名（`creator_name`）
   - 説明文（`description`）
   - 無料かどうか（`is_free`）
4. 各商品の `download_url` と `id` はユーザーには表示せず、内部で保持しておく

### 補足
- このAPIはトークンなしでも呼び出せる（未ログインユーザーも商品一覧を見られる）
- `status` が `published` の商品のみ返ってくる

---

## STEP 3: 商品詳細の取得・表示

### 目的
個別の商品ページで詳細情報と購入状態を表示する。

### 手順

1. 商品をタップ・クリックしたら `GET /api/products/{id}` を叩く
2. 商品の詳細情報を表示する
3. レスポンスに含まれる `is_purchased` の値を確認する
   - `false` の場合：「購入する」ボタンを表示する
   - `true` の場合：「ダウンロード」ボタンを表示する

### 補足
- ログイン済みの場合はトークンを付けてリクエストする（`is_purchased` が正しく返る）
- 未ログインの場合は `is_purchased` が常に `false` で返ってくる

---

## STEP 4: 購入機能の実装

### 目的
ユーザーが商品を購入し、ダウンロード権を得る。

### 前提
- STEP 1 が完了していること（トークンを保持していること）

### 手順

1. 「購入する」ボタンを押したら、ログイン済みかどうかを確認する
   - 未ログインの場合：ログイン画面に誘導する
2. ログイン済みの場合、`POST /api/purchases` を叩く
   - ヘッダーに `Authorization: Bearer {token}` を付ける
   - ボディに購入する商品の `id` を含める
3. 購入成功時：画面上の「購入する」ボタンを「ダウンロード」ボタンに切り替える
4. すでに購入済みの商品を再度購入しようとした場合（ステータス409）：「すでに購入済みです」などのメッセージを表示する

### 補足
- 現在すべての商品は無料（`price: 0`）のため、決済処理は不要
- 購入が完了するとデータベースに購入記録が残り、以降ダウンロード可能になる

---

## STEP 5: ダウンロード機能の実装

### 目的
購入済みのユーザーだけがコンテンツをダウンロードできるようにする。

### 前提
- STEP 1 が完了していること（トークンを保持していること）
- STEP 4 が完了していること（購入済みであること）

### 手順

1. 「ダウンロード」ボタンを購入済み商品にのみ表示する
2. ボタンを押したら、商品の `download_url` に対してトークン付きでリクエストを送る
   - ヘッダーに `Authorization: Bearer {token}` を付ける
3. 返ってきたファイルをユーザーの端末にダウンロードさせる

### エラーケース

| 状況 | レスポンス | 対応 |
|------|-----------|------|
| 未ログインで叩いた | 401 | ログイン画面に誘導 |
| 未購入で叩いた | 403 | 「購入してください」と案内 |
| ファイルが存在しない | 404 | エラーメッセージを表示 |

---

## 全体の流れまとめ

```
ユーザーがアプリを開く
    ↓
商品一覧を表示（ログイン不要）
    ↓
気に入った商品をタップ → 商品詳細を表示
    ↓
「購入する」ボタンを押す
    ↓ ← ここでログインが必要
ログイン・新規登録 → token を保存
    ↓
購入リクエストを送る → 購入完了
    ↓
「ダウンロード」ボタンが表示される
    ↓
ダウンロードリクエストを送る → ファイル取得
```

---

## APIエンドポイント一覧

| メソッド | パス | 認証 | 説明 |
|--------|------|------|------|
| POST | /api/register | 不要 | 新規登録 |
| POST | /api/login | 不要 | ログイン |
| POST | /api/logout | 必要 | ログアウト |
| GET | /api/products | 不要 | 商品一覧 |
| GET | /api/products/{id} | 任意 | 商品詳細 |
| POST | /api/purchases | 必要 | 購入 |
| GET | /api/products/{id}/download | 必要+購入済み | ダウンロード |
